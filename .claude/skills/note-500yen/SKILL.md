# 500円note作成スキル

500円で売れるnoteを8ステップで対話的に作成する。
各ステップでユーザーの承認を得てから次に進む。
承認は必ず **AskUserQuestionツール** を使い、選択肢付きで確認すること。

## 設定

- **成果物の保存先**: `/Users/teach/environment/app/2507/260423_AInote/drafts/{YYMMDD}_{ジャンル短縮名}/`
- **中間成果物**: 各ステップ完了時に該当ファイルを保存（`genre.md`, `persona.md`, `titles.md`, `outline.md`, `strategy.md`, `body.md`, `note-config.md`, `x-research.md`, `x-posts.md`, `review.md`）
- **Xリサーチ**: Step⑦で自動実行（ユーザーが「スキップ」と言った場合のみオフ）

## 想定スケジュール（ユーザー向けガイド）

初回実行時に以下を案内として表示する:

```
【4週間モデル】
1週目: Step①〜③（ジャンル→ターゲット→タイトル/構成）
2週目: Step④〜⑤（戦略→本文）
3週目: Step⑥〜⑦（note設定→公開→集客）
4週目: Step⑧（改善・横展開）

一気に走ることも、週単位で止めて再開することもできる。
各ステップで作った成果物はdrafts/配下にmdで保存されるので、
別日にこのスキルを再実行すれば続きから進められる。
```

## ステップ① ジャンル選定

**ジャンル選びの3基準:**
1. 悩んでいる人が多い（検索需要がある）
2. 競合が多すぎない（レッドオーシャンに突っ込まない）
3. 調べていてストレスを感じない

**定番の売れるジャンル**: 副業・節約・転職・恋愛・ダイエット・育児・英語・AI活用

**やること:**

1. ユーザーからテーマの方向性をヒアリング（副業系なのか、ライフハック系なのか 等）
2. 以下のプロンプト意図でジャンル10案を提案:

[prompt]
{ユーザーの方向性}に関連する、500円noteで売れそうなジャンルを10個提案しなさい。
各ジャンルに、なぜ売れるのか（検索需要・競合状況・リピート性）の理由を1〜2行で添えること。
上記3基準に沿って評価し、推奨度（★1〜★5）もつけること。
[/prompt]

3. 10案を表示した上で、AskUserQuestionツールで確認:
   - question: "ジャンルはどれにしますか？"
   - header: "ジャンル"
   - options（推奨上位3つ + その他）:
     - label: "{推奨1位}（推奨）"
     - label: "{推奨2位}"
     - label: "{推奨3位}"
     - label: "他の7案から選ぶ/自分で指定"

4. 確定したジャンルを `drafts/{YYMMDD}_{英語短縮名}/genre.md` に保存。

**コツの内蔵ガイド:** 「自分が詳しいもの」より「悩んでいる人が多いもの」を優先。

**承認を得てから次へ進む。**

## ステップ② ターゲット設定

「この記事、自分のことを言っている」と感じさせるのが最強の売れ方。
具体的な一人の人物像をリアルにイメージする。

**やること:**

1. 以下のプロンプト意図でペルソナ3案を生成:

[prompt]
{確定ジャンル}の500円note購入者として、具体的なペルソナを3パターン作りなさい。
各ペルソナに以下を含めること:
- 年齢・職業・年収/手取り
- 現在の悩み（3つ）
- 求めている情報の形式（何ステップ手順か、体験談ベースか、等）
- 購入を決める瞬間のトリガー（「この文を読んだら買う」の具体例）
[/prompt]

2. 3パターンを表示した上で、AskUserQuestionツールで確認:
   - question: "どのペルソナをメインターゲットにしますか？"
   - header: "ペルソナ"
   - options:
     - label: "ペルソナA"
     - label: "ペルソナB"
     - label: "ペルソナC"
     - label: "修正 or 自分で書く"

3. 確定したペルソナを `persona.md` に保存。

**コツの内蔵ガイド:** ペルソナは「noteを書く前の自分」でも大丈夫。かつての自分に向けて書くと、自然に刺さる言葉が出てくる。

**承認を得てから次へ進む。**

## ステップ③ タイトル・構成づくり

### タイトル作成

**使える型:**
- 「【〇〇円から】〇〇が△△できた方法」
- 「スキルなし・時間なしでも〇〇を達成した〇ステップ」
- 「〇〇を知らないと損する、△△の全手順まとめ」
- 「【実績公開】〇〇だけで月〇万を稼いだ話」

**やること:**

1. 以下のプロンプト意図でタイトル10案生成:

[prompt]
{ジャンル}について、{ペルソナ}向けの500円noteタイトルを10個考えなさい。
数字・実績・具体性の3要素を意識し、思わずクリックしたくなるタイトルにすること。
各タイトルの横に、使われている型（A: 金額提示 / B: ステップ数 / C: 警告 / D: 実績 / その他）を記載すること。
[/prompt]

2. 10案を表示し、AskUserQuestionで絞り込み:
   - question: "タイトル案のどれにしますか？"
   - header: "タイトル"
   - options: 上位3案 + "他の7案から選ぶ/修正"

3. 確定タイトルを `titles.md` に保存。

### 構成づくり

4. 以下のプロンプト意図で章立て生成:

[prompt]
タイトル「{確定タイトル}」のnote記事の構成を作りなさい。
{ペルソナ}が最後まで読み続けたくなる章立てにし、各章に以下を記載:
- 章タイトル
- その章で読者に伝えたい1メッセージ
- 読者の感情の動き（共感→希望→確信→行動）
章数は7〜10章を目安とする。
[/prompt]

5. AskUserQuestionで確認:
   - question: "この構成でいきますか？"
   - options: "採用" / "章を追加・削除" / "順番を変える"

6. 確定構成を `outline.md` に保存。

**承認を得てから次へ進む。**

## ステップ④ コンテンツ戦略

**3つを整理する:**
①読者の悩みを言語化
②その悩みを解決できる情報を整理
③読み終わった後に読者にどんな行動をとってほしいかを決める

**やること:**

1. 以下のプロンプト意図で戦略シート生成:

[prompt]
{ペルソナ}向け{ジャンル}noteのコンテンツ戦略を作りなさい:
1. 読者の悩み（3〜5個、具体的なセリフ形式で）
2. 記事で解決すること（悩み1つずつに対応させる）
3. 読後の行動（「今日中に〇〇する」「1週間以内に〇〇を完了する」の形式で）
[/prompt]

2. AskUserQuestionで確認:
   - question: "この戦略でいきますか？"
   - options: "採用" / "悩みを追加" / "読後行動を変更"

3. `strategy.md` に保存。

**承認を得てから次へ進む。**

## ステップ⑤ 本文の作成

**重要: AIにいきなり全部書かせようとするのがうまくいかない原因。章ごとに生成して、自分で少し手を入れるスタイルにする。**

### AI活用11のルール（本文生成時に厳守）

1. ゴールを先に書く（「〇〇のために〇〇を作って」）
2. 一度に頼むことは1つだけ
3. 読者前提を明示（「読者は〇〇な人です」）
4. 出力後に「もっと具体的に」と追う
5. NG例もセットで頼む
6. 長い資料は分割して渡す
7. 数字を具体的に指定（「200字以内で」）
8. 「なぜそう判断した？」と根拠を聞く
9. 同じ指示を3回繰り返させない（プロンプトを根本から見直す）
10. 出力は必ず自分の言葉に直す
11. 使ったプロンプトを保存しておく

### 章ごとの生成手順

1. outline.md の章ごとに、以下のプロンプト意図で本文生成:

[prompt]
以下の構成の「第{N}章: {章タイトル}」を、{ペルソナ}にわかりやすい言葉で書きなさい。
- 専門用語は使わず、具体例を入れる
- {指定文字数}字程度にまとめる
- {その章で伝えたい1メッセージ}を軸にする
- 章の最後に次の章への橋渡しを1行入れる
[/prompt]

2. 各章を順次生成し、中間ファイル `body-draft.md` に追記していく。

### AI臭を消す（必ず実施）

本文を通しで書き終えたら、以下の10選から **最低3つ** を選んで適用する:

1. **文末の重複を避ける**: 同じ語尾が3回以上連続しないよう、バリエーションを持たせる
2. **感情の起伏を表現**: 驚きや喜びなど、書き手の感情が動いている様子を言葉で表現
3. **読者への問いかけを入れる**: 質問文を文中に2箇所ほど入れる
4. **完璧すぎる構成を崩す**: 教科書のような構成ではなく、思いついた順番に話しているような構成に
5. **特定の接続詞を禁止**: 「結論として」「さらに」「重要です」を使わない
6. **個人的な体験談を挿入**: 失敗から学んだ具体的なエピソードを1つ追加
7. **語尾に揺らぎ**: 基本「です・ます」+ 時々「だ・である」や体言止めを混ぜる
8. **主観的な意見を強く出す**: 中立的な書き方を避け、こだわり・偏見込みで
9. **フィラーを挿入**: 「正直なところ」「実を言うと」などを自然に混ぜる
10. **文の長さを不揃いに**: 短い文と長い文を交互に混ぜる

### 肉付けガイド（AI下書き+人間編集の黄金比）

AIに骨子＋5,000字の下書きをガッツリ作らせて、最後に **自分の実体験・失敗談・具体的な数字** を10〜15分肉付けする。具体的には:

1. 自分の失敗体験を1〜2個入れる
2. 具体的な数字や行動を追加
3. 読者への橋渡しを入れる（「あなたの状況が違っても、この部分を少しアレンジすれば〜」）
4. 最後に「一緒にやってみよう」感を出す

3. 完成した本文をAskUserQuestionで確認:
   - question: "本文いかがですか？"
   - header: "本文確認"
   - options:
     - "採用（次へ）"
     - "特定の章を再生成"
     - "AI臭除去をもう1〜2個追加適用"
     - "トーンを変えて全部書き直し"

4. `body.md` に保存。

**承認を得てから次へ進む。**

## ステップ⑥ noteの設定

**「無料部分で読者を引き込んで、続きを有料にする」形式が一番購入率が高い。**

**基本設定:**
- 価格: **500円固定**（衝動買いされやすい価格帯）
- 無料部分: **全体の30〜40%**（「続きが気になる」と感じる手前で切る）
- 無料部分の末尾に予告文: 「この先でプロンプトを全部公開します」「実際の収益スクショはこの後」等

**サムネイル:**
- 文字多め・コントラスト強め・具体的な数字を入れる

**やること:**

1. body.md から無料部分の切れ目（全体の30〜40%地点）を提案:

[prompt]
この本文を読み、無料部分と有料部分の切れ目として最適な場所を3候補提示しなさい。
各候補について、そこで切ることで読者が感じる「続きが気になる度合い」を★1〜★5で評価すること。
[/prompt]

2. サムネイル用キャッチコピーを5つ生成:

[prompt]
500円noteのサムネイル画像に入れるキャッチコピーを5つ考えなさい。
数字・実績・ターゲットを含め、思わずクリックしたくなる表現にすること。
各コピーの狙い（共感/驚き/限定感/権威性/好奇心）を明記すること。
[/prompt]

3. AskUserQuestionで確認:
   - question: "note設定はこれでいいですか？"
   - options: "採用" / "無料部分の切れ目を変更" / "サムネコピーを再生成"

4. `note-config.md` に保存（価格・無料範囲・サムネコピー・予告文案）。

**承認を得てから次へ進む。**

## ステップ⑦ 集客・拡散（Xリサーチ込み）

**集客のメインはXとnote内部検索の2本柱。**

### 7-A. Xで最新トレンドをリサーチ（3段フォールバック）

> ユーザーが「Xリサーチスキップ」と明示した場合は 7-A をスキップして 7-B へ。

#### Phase 1: WebSearchで検索（無料）
1. 以下の検索クエリで調査:
   - `{ジャンル} note 売れ筋 site:x.com`
   - `{ジャンル} {ペルソナの悩み} Twitter`
   - ジャンル関連キーワード2〜3パターン
2. **判定:** 具体的な投稿内容を含む結果が3件以上 → Phase 2/3をスキップ

#### Phase 2: Nitterでフォールバック（無料）
3. Phase 1が不十分なら、WebFetchで以下を順に試す:
   - `https://xcancel.com/search?q={ジャンル}&f=tweets`
   - `https://twiiit.com/search?q={ジャンル}&f=tweets`
   - `https://nitter.privacydev.net/search?q={ジャンル}&f=tweets`
4. **判定:** Phase 1+2で3件以上 → Phase 3をスキップ

#### Phase 3: X API（従量課金・任意）
5. Phase 1+2でも不十分なら、`X_API_BEARER_TOKEN` 環境変数を `source ~/.bashrc 2>/dev/null; echo $X_API_BEARER_TOKEN` で確認。
6. 未設定の場合は、以下の案内を表示した上で「現在の情報で進める」に切り替える:

```
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
  X API セットアップのご案内（任意・初回のみ）
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
X APIを設定すると、Xの最新投稿をより正確にリサーチできます。
未設定でも動作しますが、リサーチ精度が下がる場合があります。

STEP 1: https://developer.x.com でアカウント作成+Bearer Token取得
STEP 2: https://console.x.com で Pay-Per-Use プランに切替
STEP 3: 「請求書作成」→「クレジット」で $5 チャージ+支出上限を $5 に設定
STEP 4: echo 'export X_API_BEARER_TOKEN="..."' >> ~/.bashrc && source ~/.bashrc
STEP 5: echo $X_API_BEARER_TOKEN で確認

```

7. 設定済みの場合は以下を実行:
```bash
curl -s -H "Authorization: Bearer ${X_API_BEARER_TOKEN}" \
  "https://api.x.com/2/tweets/search/recent?query={ジャンル}&max_results=20&tweet.fields=created_at,public_metrics,text" \

data = json.load(sys.stdin)
for t in data.get('data', []):
    m = t.get('public_metrics', {})
    print(f\"[{t.get('created_at','')}] {t['text']}\")
```

8. 統合した結果を `x-research.md` に保存:

## ステップ⑧ 改善と横展開

**収益を安定させるには横展開が鍵。一つのジャンルで勝ちパターンを見つけたら、同じ構成を別テーマに当てはめる。**

### よくある失敗5つ（振り返りチェックリスト）

本ステップでは、以下5つのチェックを行う:

- [ ] ①完璧を目指して公開が遅れていないか？（70点で出して反応を見るのが最速）
- [ ] ②AI出力をそのままコピペしていないか？（Step⑤でAI臭除去を3つ以上適用したか）
- [ ] ③1本売れなかっただけで諦めるつもりはないか？（最初の1本は必ずデータが取れる）
- [ ] ④Xでの集客を後回しにしていないか？（公開日当日からX投稿する）
- [ ] ⑤プロンプトを1回試して諦めていないか？（追加指示で結果は変わる）

**やること:**

1. 上記5つをAskUserQuestionで確認（multiSelect=true）:
   - question: "公開後に振り返るチェック項目です。今できているものをすべて選んでください"
   - options: 上記5項目
2. 未達成項目があれば、具体的なアクションを提示。

### 次テーマの提案（横展開）

3. 以下のプロンプト意図で次テーマ候補を生成:

[prompt]
今回のnote構成（ジャンル: {ジャンル}、タイトル: {タイトル}）をベースに、
同じ8ステップ型の構成で横展開できる新テーマを5つ提案しなさい。
各テーマについて:
- 新ジャンル
- リライトすべき箇所（タイトル・ペルソナ・具体例の3点）
- 推定される売れ行き（★1〜★5）
[/prompt]

4. AskUserQuestionで確認:
   - question: "次はどのテーマで書きますか？"
   - options: "次テーマA〜Cから選んで新規作成" / "今回の成果をテンプレ化して終了" / "一旦完了"

5. `review.md` にチェック結果と次テーマ案を保存。
6. テンプレ化を選んだ場合、`drafts/{YYMMDD}_{ジャンル}/TEMPLATE.md` に汎用テンプレとして保存。

## 注意事項

- **各ステップで必ず AskUserQuestionツール でユーザーの承認を得てから次に進む**（テキストで「よろしいですか？」と聞くのではなく、選択肢UIを使う）
- **中間成果物は必ず drafts/{YYMMDD}_{ジャンル}/ 配下に逐次保存する**（途中で中断→別日再開を可能にするため）
- **AI臭を消すプロンプトは本文完成時に最低3つ適用する**
- **Xリサーチはデフォルトで実行**（ユーザーが明示的に「スキップ」と言った場合のみオフ）
- **ジャンル短縮名は英数字のみ**（例: `fukugyo`, `ai-side-hustle`, `diet`）
- **価格は500円固定で案内する**（ユーザーが別の価格を指示した場合のみ変更）
- **完成した本文（body.md）を最終形として提示し、そのままnoteにコピペできる状態にする**
